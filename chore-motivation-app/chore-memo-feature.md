# 家事メモ機能(気づきのキャプチャ)

企画は [`brief.md`](./brief.md)、画面遷移全体は [`screen-flow.md`](./screen-flow.md) を参照。

## 課題

ブリーフの「気づけない」問題は「今ある家事に、今やる気になれない」という文脈で語られていたが、
それより手前に「そもそも家事表に載っていない、まだ気づいてすらいない家事がある」という課題がある。
例:「この棚、汚れやすいから週一で掃除した方がいい」と気づいた瞬間を逃すと、その気づき自体が失われる。

この気づきをその場でパッとメモでき、そのまま家事表に反映できる機能を作る。

## 決まったこと

- **入力経路**:アプリを開いて入力する。ウィジェット等でアプリを開かずにメモできる経路は、
  基本機能が実装されてから検討する(今回はスコープ外)。
- **一時的 or 繰り返す**:カレンダーの「一時的な予定」「繰り返す予定」と同じ考え方。
  - **一時的**:1回だけ家事表に載り、チェックしたら消える。
  - **繰り返す**:頻度を指定し、周期的に家事表に再度出てくる。
- **繰り返しの頻度プリセット**:毎日 / 毎週 / 隔週 / 毎月 の4種類。曜日指定・日にち指定などカレンダー
  並みの柔軟性は持たせない。
- **次に出てくるタイミングの計算方法**:**チェックした日から起算**(例:週一なら、チェックした日の
  7日後に再度出てくる)。曜日固定のような「決まったスケジュールから遅れている」感覚を出さないため。
  罰なし・プレッシャーレスの原則と合わせるための決定。
- **家事表への出方**:繰り返し項目は**期限が来たものだけ**家事表に表示する。常に全部並べておくと
  リストが伸び続け、「タスク管理のためのタスク管理」化(Habiticaの反面教師)に近づくため避ける。
- **メモ追加の入り口**:ホーム画面・家事表画面の**両方**に置く。
- **一時的項目のキャンセル導線**:必要。チェック(完了)以外に、「やっぱりやらない」で家事表から
  削除できる導線を用意する。

## 現時点でのデータイメージ(実装の詳細ではなく、要素の整理)

```
{
  text: string            // 例: "棚を拭く"
  type: "once" | "recurring"
  frequency?: "daily" | "weekly" | "biweekly" | "monthly"   // type: "recurring" の場合のみ
  lastCompletedAt: string | null
  nextDueAt: string | null // "recurring" はチェック時に (lastCompletedAt + frequency) で再計算
}
```

`once` の項目は完了したら家事表から消える(繰り返さない)。`recurring` の項目はチェック時に
`nextDueAt` を再計算し、その日まで家事表には出てこない。

## 画面遷移への影響(未反映・要確認)

この機能により、ホーム画面・家事表(チェックリスト)画面の両方に項目を追加する入り口が必要になる。
[`screen-flow.md`](./screen-flow.md) にはまだ反映していない。追加すべき候補:

- ホーム画面・家事表画面それぞれに「+ メモを追加」のような導線を置き、そこから「一時的/繰り返す」を
  選ぶ簡単な入力画面へ。
- 家事表の各項目(一時的なもの)に、チェック以外の「削除」導線を用意する。
