# 画面構成・遷移(v1)

企画は [`brief.md`](./brief.md)、パターン1のセリフ詳細は [`pattern1-dialogue.md`](./pattern1-dialogue.md) を参照。
ここでは壁打ちで固めた「画面の種類と遷移」をまとめる。**レイアウト・見た目の詳細(ワイヤーフレーム)は
まだ対象外**で、箱と矢印レベルの合意事項のみを記録する。

## 前回までの経緯(決定事項の背景)

当初案では「タイトル→(今日家事はどんな感じ?という無機質な質問画面)→ホーム」という構成だったが、
以下の理由で「キャラが自分から状態を聞く1画面」に統合した。

- キャラ不在の事務的なアンケート画面が開いた直後に挟まると、「キャラの様子が気になるから開く」という
  動機と噛み合わない。
- アプリは「今日すでに家事をしたか」を完全には自動検知できない(おみくじ経由の完了報告を通さず、
  アプリを開かずに家事を終わらせた日もあるため)。よってキャラが会話の一環として尋ね、
  ユーザーが軽く答える形は必要かつ自然な設計だった。

## 画面一覧

| # | 画面名 | 役割 |
|---|--------|------|
| 1 | タイトル | 起動画面。[始める]のみ |
| 2 | キャラ(状態確認) | キャラが「おかえりー。今日家事どんな感じ?」と話しかけ、まだ/もうやった/今日はサボる の3択を対等に提示 |
| 3 | ホーム(まだ) | ゲージ(加点表示)+キャラのセリフ(パターン1)+[家事表][おみくじ]の導線 |
| 4 | 家事表(チェックリスト) | 今日の家事項目をチェックできる共有画面。ホーム経由(閲覧/選ぶ)ともうやった経由(自己申告)の両方から来る |
| 5 | おみくじ引く画面 | [引く]でランダムに今日の家事を1つ提案 |
| 6 | おみくじ結果画面 | 「今日のミッション」表示。キャラが「完了したら教えてね」と声かけ |
| 7 | ホーム(おみくじ後) | キャラ+[家事表][完了報告]の導線 |
| 8 | 雑談 | 家事の話は一切しない、キャラの近況・気分トーク。[やっぱやる気出てきた]で復帰可能 |

## 遷移

```
タイトル
  └[始める]→ キャラ(状態確認)

キャラ(状態確認)  ※3択は対等な見た目で並べる
  ├[まだ]        → ホーム(まだ)
  ├[もうやった]   → 家事表(チェックして自己申告 → チェック分だけゲージに加点)
  └[今日はサボる] → 雑談

ホーム(まだ)
  ├[家事表]  → 家事表(チェックリスト)
  └[おみくじ] → おみくじ引く画面

家事表(チェックリスト)
  └[ホームにもどる] → ホーム

おみくじ引く画面
  └[引く] → おみくじ結果画面

おみくじ結果画面
  └[ホームにもどる] → ホーム(おみくじ後)
     (実際に家事を終えたら「完了報告」で応える想定)

ホーム(おみくじ後)
  ├[家事表]
  └[完了報告]

雑談
  └[やっぱやる気出てきた] → おみくじ引く画面(家事表を経由せず直行)
```

## 家事表(チェックリスト)画面についての合意

- 「もうやった」を選んだ場合:自己申告用にチェックリストが開き、実際にやった項目にチェックを入れる。
  チェックした項目の分だけ加点ゲージが動く。
- 「ホーム(まだ)」から[家事表]を押した場合:同じ画面を、まだやっていない項目を眺める/選ぶ用途で開く。
- 上記いずれの入り口でも同一のチェックリスト画面を共有する想定。おみくじ完了報告済みの項目は
  自動でチェック済み表示になる。
