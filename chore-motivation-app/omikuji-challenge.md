# おみくじ:マンネリ化対策(チャレンジ枠)

企画は [`brief.md`](./brief.md)、おみくじの基本フローは [`screen-flow.md`](./screen-flow.md)、
家事メモ機能は [`chore-memo-feature.md`](./chore-memo-feature.md) を参照。

## 課題

家事を続けていくと、同じような家事の繰り返しでマンネリ化してくる。おみくじで新鮮さ・刺激を
たまに混ぜて、飽きを防ぎたい。例:「作ったことない料理を1品作ってみる」のような、いつもの
家事表には載らない毛色の違う提案。

## 決まったこと

- **チャレンジ候補の出どころ**:ユーザーの家事表(メモ機能で登録したもの)とは別に、**アプリ側が
  あらかじめ用意した候補リスト**から出す。
- **出し方**:通常のおみくじ抽選プール(家事表の期限が来ている項目)に、**低確率でランダムに
  混ぜて出す**。「チャレンジ専用ボタン」のような別枠の入り口は作らない。
- **判定ロジックは持たない(v1)**:「やる気がない時に出す」のような状態判定はせず、単純な確率で
  混ぜるだけにする。将来、サボりが続いている/同じような家事ばかり選ばれているといった分かりやすい
  兆候が取れるようになったら、その時に出現率を調整する伸びしろとして残しておく。

## 現時点でのデータイメージ

```
{
  text: string       // 例: "作ったことない料理を1品作ってみる"
}
```

おみくじの抽選プールは「家事表の期限が来ている項目」+「低確率で混ぜるチャレンジ候補」の合成。

## 保留中の論点(次回以降)

- チャレンジ候補リストの具体的な中身(実際に何を何個用意するか)は別途ネタ出しが必要。
- 出現率(いったん「10回に1〜2回」くらいを仮置きしているが未確定)。
- チャレンジが当たった時、おみくじ結果画面の見せ方を通常の家事と変えるか(同じ「今日のミッション」
  表示で統一するか)。
