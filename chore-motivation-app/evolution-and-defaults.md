# 進化の仕組みとデフォルトコンテンツ

企画は [`brief.md`](./brief.md) を参照。ここでは `chore-companion/` のプロトタイプ実装で
具体化した、進化の仕組みと、家事表・おみくじのデフォルトコンテンツをまとめる。

## 進化の仕組み

ブリーフで検討事項として挙げられていた「7〜12回の間でランダムに進化」をそのまま採用した。

- 頑張り(ゲージ)が貯まり、次の進化ライン(7〜12でランダムに決定)に達すると、ステージが1つ進む。
- 進化した瞬間、専用の「レベルアップ」画面に遷移し、ステージ数を伝える。
- 進化後はゲージを0に戻し、次の進化ラインを改めて7〜12でランダムに引き直す。
- ゲージ表示自体は引き続き「これまでの頑張り、◯個たまってるよ」という加点表現のみで、
  「あと◯回で進化」という逆算表現は出さない。

キャラの見た目・名前が未定なため、レベルアップ画面は仮の絵文字(卵→ひよこ→鳥、のような並び)を
ステージ数に応じて出しているだけで、実際のキャラクターが決まったら差し替える前提。

## デフォルトの家事表(一人暮らし向け)

初回起動時に入っている繰り返し項目。一時的な項目はデフォルトでは入れず、ユーザーがメモ機能で
都度追加する想定。

- 毎日:使った食器を洗う / シンクの水気を拭く
- 毎週:洗濯機を回す / 掃除機をかける / お風呂を洗う / トイレを掃除する
- 隔週:シーツ・枕カバーを替える
- 毎月:冷蔵庫の中を整理する / 換気扇のフィルターを掃除する

## おみくじのチャレンジ候補(デフォルト)

[`omikuji-challenge.md`](./omikuji-challenge.md) で決めた「低確率で混ぜる」チャレンジ枠の、
現時点での候補リスト。

- 作ったことない料理を1品作ってみる
- 引き出しを1つだけ整理してみる
- 捨てるか迷ってるものを1個だけ決める
- 普段拭かない場所を1箇所だけ拭いてみる
- クローゼットの中身を1段だけ見直す
- 使ってない調味料を使い切るレシピを探してみる
- 玄関の靴を整理してみる
- 1年以上使ってないものを1つだけ見つけてみる

どちらのリストも実装上は `chore-companion/lib/store.ts` にあり、中身の増減は容易にできる。
